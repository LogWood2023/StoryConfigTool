## 各 Type 详细说明

### Type=1 · 跳到指定帧

**功能**：跳过中间帧，直接跳转到指定演出帧。

**参数格式**：`FrameId`（格式：`GroupId + 帧序号`，如 `100105` = 第1001组第5帧）

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `StoryFrame.xlsx` | 跳转目标帧的定义 |

---

### Type=2 · 触发指定剧情组

**功能**：在当前剧情中插入/跳转到另一个剧情组（常用于选项分支）。

**参数格式**：`StoryGroupId`

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `StoryGroup.xlsx` | 被触发的剧情组定义 |
| `StoryFrame.xlsx` | 当前帧的 Behavior 字段引用此行为 |

**注意**：配合 `StoryFrame.Behavior` 的选项权重格式使用，可实现分支剧情。

---

### Type=3 · 领取任务

**功能**：玩家领取指定任务（加入任务列表）。

**参数格式**：`TaskId`

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `Task.xlsx` | 领取的任务定义；`AcceptCondition` 控制是否可接 |
| `Condition.xlsx` | 领取条件的判断 |

---

### Type=4 · 完成任务

**功能**：玩家完成指定任务（任务从任务列表移除，触发完成奖励）。

**参数格式**：`TaskId`

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `Task.xlsx` | 完成的任务定义；`Reward` 字段配置完成奖励 |
| `Condition.xlsx` | `CompleteCondition` 判断完成条件 |

---

### Type=5 · 获得状态

**功能**：将指定 Mark 标记为"已获得"（Mark值设为1）。

**参数格式**：`MarkId`

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `Mark.xlsx` | 定义标记状态 |
| `Condition.xlsx` | 通过 Condition 引用 Mark 值 |

---

### Type=6 · 消除状态

**功能**：将指定 Mark 标记为"未获得"（Mark值设为0）。

**参数格式**：`MarkId`

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `Mark.xlsx` | 定义标记状态 |
| `Condition.xlsx` | 通过 Condition 引用 Mark 值 |

---

### Type=7 · 获得道具

**功能**：给予玩家道具（通过 ObtainItem 表配置）。

**参数格式**：`Obtain id`（ObtainItem 表的 id）

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `ObtainItem.xlsx` | 定义获得的道具种类、数量 |
| `Item.xlsx` | 道具基础定义 |

**注意**：不直接填 ItemId，而是先在 ObtainItem 表配置好再引用其 id。

---

### Type=8 · 上交道具

**功能**：让玩家上交/消耗指定道具（通过 GiveItem 表配置）。

**参数格式**：`GiveItem id`（GiveItem 表的 id）

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `GiveItem.xlsx` | 定义上交的道具种类、数量 |
| `Item.xlsx` | 道具基础定义 |

**注意**：与 Type=7 类似，通过 GiveItem 表配置具体道具。

---

### Type=9 · 结束当前剧情组

**功能**：立即结束当前剧情组，不再继续后续帧。

**参数格式**：_无参数_

**涉及表格**：无（纯控制行为）

---

### Type=10 · 进入战斗

**功能**：切入战斗场景，战斗结束后返回剧情。

**参数格式**：`Stage id`（Stage 表的 id）

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `Stage.xlsx` | 战斗关卡定义（怪物、胜利条件等） |
| `Monster.xlsx` / `MonsterGroup.xlsx` | 战斗中的怪物配置 |

**注意**：战斗胜利/失败后，剧情继续（由 Stage 配置决定后续行为）。

---

### Type=11 · 跳到下一帧

**功能**：自动跳到当前帧的 NextFrame（无需指定参数，读取 StoryFrame.NextFrame）。

**参数格式**：_无参数_

**涉及表格**：
| 表格 | 用途 |
|------|------|
| `StoryFrame.xlsx` | 通过 NextFrame 字段确定跳转目标 |

**注意**：与 Type=1 的区别是无需指定目标帧，自动读 NextFrame。

---

**参数格式**：`事件id,显示状态`
- 显示状态：`1` = 显示，`0` = 隐藏

**涉及表格**：

| `MapEvent.xlsx` | 事件的 `IsShow`/`IsHide` 字段被修改 |

### Type=18 · 增加动态对象的好感度

**功能**：增加（或降低）NPC或玩家对彼此的好感度。

**参数格式**：`增加对象,增加数值`
- 增加对象：
  - `1` = 增加NPC对自己的好感度
  - `2` = 增加自己对NPC的好感度
  - `3` = 两边都增加
- 增加数值：填负表示降低好感度

**涉及表格**：无（好感度直接存储在游戏里，不存储在表格中）

**注意**：好感度没有上限/下限。

## 附加行为（AdditionBehavior）类型说明

`StoryBehavior.AdditionBehavior` 字段可以配置多个附加行为，格式与 Behavior 类似，支持的类型包括：

| 附加行为类型 | 功能 | 参数格式 |
|---------------|------|----------|
| **3** | 领取任务 | `TaskId` |
| **4** | 完成任务 | `TaskId` |
| **12** | 改变事件显示状态 | `事件id,显示状态` |
| **17** | 设置 Condition 值 | `ConditionId,值` |
| **21** | 设置剧情选项结果 | `0` |
| **24** | 强制结束/刷新 | _无参数_ |
| **29** | 触发特殊技能效果 | `技能类型,技能Id` |

**示例**：
```
AdditionBehavior = "3,1011;3,1012;4,1009;12,101007,1;12,101008,1"
  → 附加行为：
    ① 领取任务1011
    ② 领取任务1012
    ③ 完成任务1009
    ④ 显示事件101007
    ⑤ 显示事件101008
```

## StoryFrame.Behavior 字段格式详解

在 `StoryFrame.xlsx` 的 `Behavior` 列中引用 StoryBehavior：

### 格式一：强制执行（无选项）

```
Behavior = "BehaviorId1,BehaviorId2,BehaviorId3"
```
所有行为按顺序强制执行，无玩家选择。

### 格式二：选项式触发（有选择）

```
Behavior = "0,选项数下限,选项数上限;BehaviorId1,权重1;BehaviorId2,权重2;..."
```

| 部分 | 说明 |
|------|------|
| `0` | 固定前缀，表示"选项模式" |
| `选项数下限,选项数上限` | 随机出现几个选项（如 `1,2` = 出现1~2个选项） |
| `BehaviorId,权重` | 每个选项对应的行为和出现权重 |

**示例**：
```
Behavior = "0,1,2;20003,50;20004,50"
```
