## 一、表格文件命名规则

1. **策划案文件命名**：`玄幻项目功能策划-[功能模块].xlsx`
   - 例如：`玄幻项目功能策划-丹药.xlsx`、`玄幻项目功能策划-宗门.xlsx`
   
2. **补充/修正文件命名**：`[功能模块]修正美术需求.xlsx`、`[功能模块]补充.xlsx`

3. **每个xlsx文件通常包含**：
   - 一个主Sheet（功能描述 + 原型表定义）
   - 若干空Sheet（Sheet1/Sheet2/Sheet3，通常为空，可能作为草稿占位）

## 二、原型表（配置表 Schema）定义规则

### 2.1 字段定义格式

每个字段使用统一的描述格式（两种变体）：

**格式A（最常见）**：
```
→中文名/英文名/类型
```

**格式B**：
```
→英文名/中文名/类型
```

**示例**：
```
→id/Id/int
→名字/Name/string
→描述/Dsc/string
→任务类型/Type/int
→任务可接条件/AcceptCondition/int[][]
```

### 2.2 数据类型

| 类型标识 | 含义 | 说明 |
|---------|------|------|
| `int` | 整数 | 单值 |
| `string` | 字符串 | 单值文本 |
| `int[]` | 整数数组 | 一维数组，如 `[1,2,3]` |
| `int[][]` | 二维整数数组 | 通常表示配对的键值对列表 |
| `string[]` | 字符串数组 | 一维字符串数组 |
| `string[][]` | 二维字符串数组 | 如对话文本组 |

### 2.3 通用字段（几乎所有原型表都有的字段）

| 字段 | 格式 | 说明 |
|------|------|------|
| `id` | `→id/Id/int` | 唯一标识，所有表的第一个字段 |
| `名字` | `→名字/Name/string` | 显示名称 |
| `描述` | `→描述/Dsc/string` | 文本描述 |

## 三、数据填写格式规则

### 3.1 条件（Condition）填写格式

这是使用最广泛的数据格式，出现在几乎所有表中：

```
格式：(条件id, 最小值, 最大值)
```

**规则**：
- `(1001, 0, 0)` → 条件1001的值必须等于0
- `(1001, 1)` → 条件1001的值必须大于等于1（省略最大值）
- `(1001, 1, 3)` → 条件1001的值必须在1到3之间（闭区间）

**二维数组写法**：多个条件用括号分隔
```
(1001, 1)(2001, 0, 0)(3005, 5)
```

### 3.2 权重/概率填写格式

```
格式：item_id, 权重
```

**示例**：
- `1, 50; 2, 30; 3, 20` → 三个条目分别权重50、30、20
- 用分号 `;` 分隔多个条目
- 用逗号 `,` 分隔条目内部参数

### 3.3 道具/奖励填写格式

```
格式：(类型, 参数)
```

**常见类型**：
- 类型1 → 给道具和buff：`(1, 道具id, 数量)`
  - 可以填多个道具：`(1, 1001, 5, 2001, 1)`
- 类型2 → 触发剧情：`(2, 剧情组id)`

**完整示例**：`(1, 1001, 5, 2001, 1)(2, 5001)`

### 3.4 数组分隔符

| 分隔符 | 用途 | 示例 |
|--------|------|------|
| `,` | 同一组内参数分隔 | `宗门id, 概率权重` |
| `;` | 不同组之间的分隔 | `item id, 数量; item id, 数量` |
| `()` | 每个数据组的外壳 | `(1, 50)(2, 30)` |

### 3.5 二维数组格式（int[][]）

当字段标注为 `int[][]` 类型时，表示一组多组配对数据：

```
格式：参数1, 参数2; 参数1, 参数2; ……
```

**示例**：
- 境界配置：`1, 50; 2, 30; 3, 20` → 境界1权重50，境界2权重30，境界3权重20
- 天赋配置：`1, 0; 2, 0; 3, 3002` → 境界1天赋随机，境界3天赋固定为3002

### 7.3 属性公式体系

属性值计算有三级结构：

```
属性名 → 属性加成 → 属性减免
```

核心公式：
```
最终属性 = Σ基础属性 * max(下限系数, 1 + Σ加成) * (1 + Σ升华)
```

特殊公式：
- 防御减免率：`k * 最终防御 / (10000 + k * 最终防御)`（k随等级动态变化）
- 五行抗性：`s * 最终五行防御 / (10000 + s * 最终五行防御)`

## 十一、配置规则要点汇总

1. **所有原型表必须首字段为 `id/Id/int`**
2. **字段描述统一使用 `→中文名/英文名/类型` 格式**
3. **条件字段统一使用 `(条件id, 最小值, 最大值)` 格式**
4. **多组数据使用 `;` 分隔，组内使用 `,` 分隔**
5. **品质统一1-6级（白绿蓝紫橙红）**
6. **枚举值的0通常代表"无/空/默认"**
7. **权重随机时，id=0通常代表"随机选择"**
8. **战斗外功能通常由Condition控制解锁**
9. **技能配置使用Category/Param体系，参数位置固定**
10. **剧情使用三段式结构：剧情组→演出帧→剧情行为**
11. **常量表（Constant.xlsx）用于存储全局配置参数**
12. **道具的附加参数存储在Item表的"自定义参数"字段中**

## 十二、StoryFrame 核心字段规则

### 12.1 AdditionalText（性别变量 / {0} 占位符）

Text列含 `{0}` 时，代表按玩家性别显示不同文本。格式：
```
1+男性文本|2+女性文本
```
例：Text=`我其实是个{0}` AdditionalText=`1+男的|2+女的`
→男玩家看到"我其实是个男的"，女玩家看到"我其实是个女的"

### 12.2 AdditionPar（角色站位/表情/动效）

格式：`角色id,左右,亮暗,表情id,效果`
最多2组，用 `;` 分隔。

| 位置 | 参数 | 说明 |
|------|------|------|
| 1 | 角色id | Npc.xlsx的ID，1=主角 |
| 2 | 左右 | 1=左 2=右 |
| 3 | 亮暗 | 1=亮(说话者) 2=暗(沉默) |
| 4 | 表情id | 0=默认 1001=伤心 1007=开心 |
| 5 | 效果 | 0=无 1=放大 2=跳动 3=突一下 |

**规则**：亮=1的角色数应为1（此人说话）或0（旁白）。两个角色都亮=1 → 报错。
