# 添加新剧情 —— 完整操作指南

> 基于 `e:\work\public\data\PublicTables\` 下实际配置文件的分析。

---

## 一、剧情系统架构概览

本游戏剧情采用 **三段式结构**：**剧情组 → 演出帧 → 剧情行为**

```
┌──────────────────────────────────────────────────┐
│  StoryGroup（剧情组）                              │
│  ├─ 定义一段完整剧情                                │
│  ├─ 控制触发条件 & 完成标记                          │
│  └─ 由多帧组成                                     │
│                                                    │
│  StoryFrame（演出帧）—— 一帧 = 一个画面              │
│  ├─ 对话文本、美术图、声音                           │
│  ├─ 可挂载行为（给道具、进战斗、跳转……）               │
│  └─ 通过 FrameId 串联成链                           │
│                                                    │
│  StoryBehavior（剧情行为）                          │
│  ├─ 注册可复用的行为模板                             │
│  ├─ 类型包括：跳帧、触发剧情、领任务、进战斗、传送……    │
│  └─ 被 StoryFrame.G 引用                           │
└──────────────────────────────────────────────────┘
```

---

## 二、涉及配置表一览

| 表文件 | 作用 | 修改/新增 |
|--------|------|----------|
| **Condition.xlsx** | 定义条件（触发条件、完成条件、前置检查） | 可能需要新增 |
| **Mark.xlsx** | 标记状态（0/1），记录"是否做过某事" | 可能需要新增 |
| **StoryGroup.xlsx** | 剧情组 —— 定义整段剧情 | **必须新增 1 行** |
| **StoryFrame.xlsx** | 演出帧 —— 逐帧定义对话和画面 | **每帧新增 1 行** |
| **StoryBehavior.xlsx** | 剧情行为 —— 可复用的行为模板 | 视需要新增 |
| **Stage.xlsx** | 战斗关卡 | 如涉及战斗，需要引用已有 Stage |
| **GiveItem.xlsx** | 上交道具配置 | 如涉及上交道具 |
| **ObtainItem.xlsx** | 获得道具配置 | 如涉及给道具 |
| **Task.xlsx** | 任务 | 如剧情需配合任务 |

## 三、Step by Step 操作流程

### Step 0: 前期规划

在动手配置前，确定以下信息：

1. **剧情在什么时机触发？**
   - 地图上点击NPC？ → 非强制剧情，由NPC交互触发
   - 走到某个格子？ → 地图事件(MapEvent)触发
   - 完成某个任务？ → 任务触发
   - 达到某个条件？ → 条件触发
   - 其他剧情结束后？ → 上一个剧情组完成后的连锁

2. **剧情有几帧？** 每帧说什么话？配什么图？

3. **需要哪些行为事件？** 例如：给道具、进入战斗、传送到某地……

4. **剧情是否只触发一次？** 如果是，需要 Condition + SelfAddCondition 控制

---

### Step 1: 创建条件（Condition.xlsx）『如需要』

如果现有 Condition 表已有合适的条件（例如 `10001=剧情组2001已完成`），可跳过此步骤。

**Condition 表字段**：

| 列 | 字段 | 说明 |
|----|------|------|
| A | Id | 条件ID（例：10010） |
| B | Name | 条件名称（例：新手剧情完成） |
| C | PreOrConditions | 「或」前置条件组，格式：`(条件Id,最小值,最大值)` |
| D | PreConditions | 「且」前置条件，格式：`(条件Id,最小值,最大值;条件Id,最小值,最大值)` |
| E | Condition | 条件类型枚举（见下方） |
| F | ConditionParams | 条件参数 |
| G | ExtConditionParams | 扩展条件参数 |
| H | DataType | 数据类型：0=累加 1=当前值 2=最大值 3=最小值 |
| I | RefreshType | 刷新类型：0=不刷新 1=按天 2=按周 3=按月 |
| M | Client | 是否可被客户端修改：1=是 |

**常用 Condition 枚举值**（E列）：
- `0` —— 必定满足（常用于或判断）
- `1` —— 特殊条件（由前置条件 PreConditions 决定含义）
- `2` —— 角色大境界等级
- `6` —— 时间年数
- `7` —— 获得指定任务状态
- `8` —— 玩家拥有指定装备数量
- `9` —— 玩家拥有指定道具数量
- `10`—— 指定 Stage 的胜利次数

**新建条件示例**：
```
A=10010  B=完成新手剧情  E=1  M=1
```
（通过 PreConditions 关联其他已有条件）

---

### Step 2: 创建标记状态（Mark.xlsx）『如需要』

如果需要记录"玩家是否已经看过这个剧情"或"是否已经上交了某道具"，需要新增 Mark：

| 列 | 字段 | 说明 |
|----|------|------|
| A | MarkId | 标记ID（例：501） |
| B | Name | 名字（例：已击败守门妖兽） |

每个角色的每个 MarkId 都是一个 0 或 1 的值。

---

### Step 3: 创建剧情行为（StoryBehavior.xlsx）『如需要』

如果现有的 StoryBehavior 已经有你需要的类型（如给道具、进战斗），可以复用已有ID。否则新增：

| 列 | 字段 | 说明 |
|----|------|------|
| A | BehaviorId | 行为ID（例：20001） |
| B | Name | 名字 |
| C | ShowName | 界面按钮显示的文本 |
| D | Type | **行为类型**（见下方） |
| E | Markid | 标记状态id |
| F | Parameter | 行为参数 |
| G | NotShowCondition | 不显示按钮的条件 |
| H | AdditionBehavior | 附加行为（类型3,4,5,6,7,12,14,16,17,18,19,21,22,24,25,28,29可用） |

**行为类型（Type）枚举**：

| Type | 功能 | 参数填写 |
|------|------|---------|
| 1 | 跳到指定帧 | FrameId，格式：`组id + 帧序号`（如 100105 = 第1001组的第5帧） |
| 2 | 触发指定剧情组 | StoryGroupId |
| 3 | 领取任务 | TaskId |
| 4 | 完成任务 | TaskId |
| 5 | 获得状态 | MarkId |
| 6 | 消除状态 | MarkId |
| 7 | 获得道具 | ObtainItem 表的 id |
| 8 | 上交道具 | GiveItem 表的 id |
| 9 | 结束当前剧情组 | 无参数 |
| 10 | 进入战斗 | Stage id |
| 11 | 跳到下一帧 | 无参数 |
| 12 | 改变指定事件显示状态 | 事件id,显示状态(1=显示 0=隐藏) |
| 13 | 传送到指定地点并结束剧情组 | 地图id,坐标x,坐标y |
| 14 | 删除事件 | 事件id（空=删除触发自己剧情的事件） |
| 15 | 切磋 | Stage id |

---

### Step 4: 创建剧情组（StoryGroup.xlsx）『必须』

在 **StoryGroup** 表末新增一行：

| 列 | 字段 | 说明 | 示例 |
|----|------|------|------|
| A | StoryGroupId | 剧情组ID（唯一，整段剧情的标识） | `3001` |
| B | Name | 名字 | `第一章-遇到老乞丐` |
| C | TriggerCondition | 触发条件，格式：`条件id,值` | `10001,0` |
| D | SelfAddCondition | 自增条件id，剧情完成后 `条件值+=1` | `10001` |
| E | Notes | 备注 | `主角第一次进入荒原时触发` |
| F | RandomStoryGroup | 随机剧情组，格式：`剧情组id,权重;……` （为空则直接触发本组） | （留空） |

**TriggerCondition 的隐含语义**：
- `10001, 0` → ConditionId=10001 的值必须 **等于** 0（从未触发过）
- `10001, 1` → ConditionId=10001 的值必须 **>=** 1
- 为空 → 无条件触发

**常见模式**：触发条件 = `X,0`，自增条件 = `X` → 表示"只触发一次"

---

### Step 5: 创建演出帧（StoryFrame.xlsx）『每帧一行』

**StoryFrame 表字段**：

| 列 | 字段 | 说明 | 示例 |
|----|------|------|------|
| A | FrameId | 演出帧ID（建议格式：GroupId + 两位数序号） | `300101` |
| B | Name | 名字 | `第1帧-老乞丐倒在地上` |
| C | Type | **帧类型** | `1` |
| D | GroupId | 所属剧情组id | `3001` |
| E | NextFrame | 下一个演出帧id | `300102` |
| F | IsLastFrame | 是否最后一帧，若是则填：`[下一个剧情组id]` 或空 | （留空或填下一个GroupId） |
| G | Behavior | 行为参数 | `1,20001;2,20002` |
| H | Picture | 美术资源路径 | `Story/HuangYuan` |
| I | Picture_2 | 美术资源_2 | |
| J | SoundId | 声音id | |
| K | Text | 文本（对话内容） | `前方路边蜷缩着一个衣衫褴褛的老乞丐……` |
| L | AdditionalText | 附加文本（type=6时填感谢页链接） | |
| M | AdditionPar | 附加参数 | 见下方 |
| N | Summary | 剧情梗概（策划备注用） | |
| O | TransitionEffect | 转场效果 | |

**帧类型（Type）枚举**：

| Type | 说明 |
|------|------|
| 1 | 对话 |
| 2 | 图文弹出框（大） |
| 3 | 全屏图文/动画 |
| 4 | 镜头移动 |
| 5 | 图文弹出框（小） |
| 6 | 玩家感谢页 |
| 7 | 标题页 |

**AdditionPar（附加参数）填写格式**：
- Type=1（对话）：`角色id, 左右位置, 亮/暗, 附加表情, 附加效果`
  - 左右位置：1=左 2=右
  - 附加效果：1=放大 2=上下震动 3=往前突一下
  - 示例：`1,1,1,1001,1;2,2,2,1001,2`
- Type=2/3（全屏图文）：`1=图片 / 2=动画 / 3=视频`
- Type=7（标题页）：`持续时间`

**Behavior（行为）填写格式**：
- 如果所有选项都是强制行为（不出现选择框）：`BehaviorId, BehaviorId, ...`
  - 示例：`20001, 20002` → 同时执行行为20001和20002
- 如果出现选择项：
  - 第一个数组：`0, 选项数下限, 选项数上限`
  - 后续：`BehaviorId, 出现权重; BehaviorId, 出现权重`
  - 示例：`0,2,3;20001,50;20002,30;20003,20`

**帧链串联方式**：通过 `NextFrame`（E列）指向下一帧的 FrameId，最后一帧的 `IsLastFrame`（F列）标记结束并可选跳转到下一个剧情组。

---

### Step 6: 关联触发源

剧情组创建好之后，需要让它在合适的时机被触发。常见触发方式：

**方式1：NPC 交互触发**
- 在 `Npc.xlsx` 中该 NPC 的剧情配置字段填入 StoryGroupId

**方式2：任务触发**
- 在 `Task.xlsx` 的 AcceptTrigger / CompleteCondition 中配置：
  - `类型2=剧情，参数=剧情组id`

**方式3：地图事件触发**
- 在 `MapEvent.xlsx` 中配置事件，关联 StoryGroupId

**方式4：上一个剧情组完成触发**
- 在 StoryGroup 的 `IsLastFrame`（F列）或 Behavior 中使用 Type=2（触发指定剧情组）

## 四、完整示例：添加一章「遇到老乞丐」剧情

假设需求：
- 玩家第一次进入荒原地图时，自动触发
- 第1帧：老乞丐倒在地上，对话
- 第2帧：老乞丐请求帮助，出现选项「帮助他」/「无视」
- 选择「帮助他」→ 获得道具「破旧的令牌」(itemId=5001)
- 选择「无视」→ 结束剧情
- 只触发一次

### 1. 检查/创建 Condition
```
A=10010  B=完成遇到老乞丐剧情  E=1  M=1
```

### 2. 创建 StoryGroup（剧情组）
```
A=3001  B=遇到老乞丐  C=10010,0  D=10010  E=荒原首次进入触发
```

### 3. 创建 StoryBehavior（剧情行为）

```
A=20101  B=给予破旧令牌  C=收下令牌  D=7  F=[ObtainItem表id]
A=20102  B=结束剧情       C=离开      D=9
A=20103  B=帮助老乞丐      C=帮助他    D=7  F=[ObtainItem表id]
```

### 4. 创建 StoryFrame（演出帧）

**帧1** —— 发现老乞丐：
```
A=300101  B=第1帧  C=1(对话)  D=3001  E=300102
K= 前方路边蜷缩着一个衣衫褴褛的老乞丐，看起来已经奄奄一息……
H= Story/HuangYuan
M= 1,2,1  （角色1在右侧，亮）
```

**帧2** —— 出现选项：
```
A=300102  B=第2帧  C=1(对话)  D=3001  E=300103
K= 老乞丐用虚弱的声音说：「年轻人……能……帮帮老夫吗……」
G= 0,1,2;20103,50;20102,50  （随机出现1~2个选项，两个行为各50%权重）
M= 1,2,1
```

**帧3** —— 结算帧：
```
A=300103  B=第3帧  C=1(对话)  D=3001
F=        （最后一帧，留空）
K= 不知这老乞丐是何许人也……但他给的令牌似乎有些来历。
G= 20101    （强制执行：给道具）
```

### 5. 关联触发
在荒原地图的 MapEvent 中配置此 StoryGroupId，或直接在游戏启动的初始化逻辑中检查 Condition 10010=0 时触发。

## 五、注意事项

1. **FrameId 建议包含 GroupId**：如 `300101` 表示 GroupId=3001 的第 `01` 帧，方便阅读和管理。

2. **只触发一次的剧情**：TriggerCondition 填 `条件id, 0`，SelfAddCondition 填同一个 `条件id`。剧情最后一帧结束后，程序会将条件值 +1。

3. **条件值含义**：
   - `X, 0` → 等于0
   - `X, 1` → 大于等于1
   - `X, 1, 5` → 1到5之间

4. **剧情行为需要在 StoryBehavior 表中注册**，然后在 StoryFrame.G 中通过 BehaviorId 引用。

5. **给道具/上交道具**不直接在 StoryFrame 中配置具体 ItemId，而是在 ObtainItem.xlsx / GiveItem.xlsx 中配置，Behavior 引用它们的 id。

6. **剧情中断恢复**：如果剧情涉及数据变更（如上交道具），需要配合 Mark 表记录"已经做过这个操作"，中断重来时检查 Mark 跳过已执行的步骤。

7. **随机剧情组**：StoryGroup 的 RandomStoryGroup 字段可以为一段剧情配置多种变体（如切磋赢了走剧情1，输了走剧情2），在 StoryGroup/Sheet4 中有配置示例。

8. **转场效果**（StoryFrame.O列）支持淡入淡出和全屏抖动两种效果。
