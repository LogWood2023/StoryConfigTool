# 任务一览 工作流

## 整体流程

```
打开入口 → 加载数据 → 浏览列表 → 选择任务 → 编辑字段 → 保存暂存 → 导出文件
```

---

## 1. 打开入口

触发方式：
- 条件详情中点击任务引用链接（`onRefClick('Task', id)`）
- 右侧一览入口栏点击"任务"按钮（`openOverview('Task')`）

打开后执行 `openTaskModal(taskId)`：
- 显示模态窗口
- 并行请求：`/api/task-list`、`/api/task-group-options`、`/api/realm-level-options`
- 缓存任务列表到 `taskListCache`（避免重复请求）
- 渲染左侧任务列表
- 如果传入了 `taskId`，自动加载该任务详情

## 2. 浏览与搜索

- 左侧列表显示所有任务（ID + 名字）
- 搜索框支持按 ID 或名字过滤
- 点击列表项切换右侧详情
- 新增按钮（＋）创建空白任务

## 3. 编辑字段

编辑区域由 `renderTaskDetail(d)` 渲染，所有修改记录到 `taskEditData.changes` 对象中。

## 4. 保存暂存

点击"保存"按钮执行 `saveTaskChanges()`：
- 检查是否有修改
- ID 重复检测
- 将修改合并到 `pendingChanges[Task_原始ID]`
- 记录撤回栈（最多10步）
- 更新左侧列表中的 ID/名字显示
- 如果从引用链接打开，更新源引用文字

## 5. 撤回

`Ctrl+Z` 从 `undoStack` 弹出上一步状态，恢复 `pendingChanges` 并刷新详情。

## 6. 导出

工具栏"导出"按钮执行 `exportAll()`：
- 将 `pendingChanges` 发送到 `POST /api/export`
- 后端写入 `app/导出文件/YYYYMMDD_HHMMSS/` 目录
- 成功后清空 `pendingChanges` 和 `undoStack`

## 可输入部分的种类

任务详情中的输入控件分为以下 **6 种类型**：

### 类型一：纯文本输入

```html
<input class="task-editable" value="..." data-field="xxx" onchange="taskFieldChanged(this)" />
```

适用字段：
| 字段 | data-field | 说明 |
|------|-----------|------|
| ID | `id` | 任务唯一标识 |
| 名字 | `name` | 任务名称 |
| 任务描述文本 | `dsc` | 带 `dsc-text` 样式，聚焦时显示镜像预览面板 |

### 类型二：数字输入

```html
<input class="task-editable" type="number" value="..." data-field="xxx" onchange="taskFieldChanged(this)" />
```

适用字段：
| 字段 | data-field | 说明 |
|------|-----------|------|
| 倒计时/天 | `countdown` | 任务限时天数 |
| 分组权重 | `groupWeight` | 公会/城镇任务的权重 |
| 好感度 | `addFavor` | 完成后增加的好感度 |
| 下一任务ID | `nextTask` | 完成后自动领取的任务 |
| 条件min/max | `acceptCondition_N_min/max` | 条件判定范围 |
| 回收道具数量 | `removeItem_N_count` | 回收道具个数 |
| 奖励数量 | `reward_N_count` | 奖励道具个数 |

### 类型三：下拉选择

```html
<select class="task-select" data-field="xxx" onchange="...">
    <option value="k" selected>v</option>
</select>
```

适用字段：
| 字段 | data-field | 选项来源 |
|------|-----------|---------|
| 任务类型 | `type` | TASK_TYPES（主线/支线/宗门/城镇/奇遇） |
| 任务子类型 | `subType` | TASK_SUBTYPES（捉拿叛徒/击杀仇敌/...） |
| 任务分组 | `group` | taskGroupOptions（API动态获取） |
| 境界等级 | `realmLevel` | realmLevelOptions（API动态获取） |
| 任务难度 | `difficulty` | DIFFICULTY_MAP（容易~地狱） |
| 是否可重复 | `isRepeat` | 0=否 / 1=是 |
| 是否隐藏 | `isHide` | 0=否 / 1=是 |
| 是否可原地交还 | `allowDirectReturn` | 0=否 / 1=是 |
| 触发器类型 | 触发器内部 | 给道具/触发剧情/生成事件/修改计数器 |

### 类型四：引用ID输入（editableRef）

```html
[<input class="ref-id-input" type="text" value="123" data-table="Table" data-ref-field="xxx" onblur="onRefIdChange(this)" />]
<span class="ref-name-ok">名称</span>
```

特点：
- 输入框显示关联表的 ID
- 右侧 span 自动显示对应名称（通过 `/api/lookup-name` 查询）
- 名称可点击跳转到对应一览
- 查询失败时显示红色"未找到"

适用字段：
| 关联表 | 使用场景 |
|--------|---------|
| Condition | 可接条件、完成条件、自增条件、触发器中的计数器 |
| Item | 回收道具、奖励道具、触发器中的给道具 |
| MapEvent | 追踪点、触发器中的生成事件 |
| Story | 触发器中的触发剧情 |
| Task | 下一任务（nextTask） |

### 类型五：复合列表组件

由专用渲染函数生成，包含多个子输入 + 删除按钮 + 新增按钮：

| 组件 | 渲染函数 | 子元素组成 |
|------|---------|-----------|
| 条件列表 | `renderConditionList` | 引用ID + min数字 + max数字 + 🗑删除 |
| 触发器列表 | `renderTriggerField` | 类型下拉 + 类型对应内容 + 🗑删除 |
| 回收道具 | `renderRemoveItem` | 引用ID + 数量数字 + 🗑删除 |
| 奖励列表 | `renderReward` | 引用ID + 数量数字 + 🗑删除 |
| 自增条件 | `renderSelfAddCondition` | 引用ID + 数值 + 🗑删除 |

每个列表末尾有"＋"按钮调用 `addFieldEntry(fieldName)` 新增条目。

### 类型六：追踪点组件

```
renderTrackField(track, fieldName)
```

特点：
- 显示 MapEvent 引用ID + 事件名
- 附带地图坐标信息（地图ID-x,y）
- 有 🗑 按钮可清除
- 不可直接新增，需通过 MapEvent 关联

## 输入种类总结

| # | 种类 | 控件形式 | 交互方式 |
|---|------|---------|---------|
| 1 | 纯文本输入 | `<input>` | 直接键入文字 |
| 2 | 数字输入 | `<input type="number">` | 键入数字 |
| 3 | 下拉选择 | `<select>` | 从预设选项中选择 |
| 4 | 引用ID输入 | `<input>` + 名称span | 输入ID后自动查询显示名称 |
| 5 | 复合列表 | 多个子控件组合 | 增删条目 + 编辑子字段 |
| 6 | 追踪点 | 引用 + 坐标显示 | 编辑引用ID / 清除 |
