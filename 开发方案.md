# 剧情配置工具 —— 完整开发方案

> 项目位置: `e:\work\剧情配置工具\`
> 技术栈: React 18 + TypeScript + Electron + ReactFlow + Ant Design + Zustand

---

## 一、项目概述

### 1.1 目标
开发一款桌面工具，策划只需将剧情Excel配置文件放入指定文件夹，工具自动解析并：
- 📊 **可视化展示**剧情流程（节点+连线+分支）
- ✏️ **直接编辑**节点内的台词/角色/表情/背景
- 📤 **一键导出**修改后的配置回Excel

### 1.2 技术选型

| 层级 | 技术 | 作用 |
|------|------|------|
| 桌面壳 | **Electron 33** | 打包为Windows exe，访问本地文件系统 |
| UI框架 | **React 18 + TypeScript** | 组件化开发，类型安全 |
| 流程图 | **@xyflow/react** (ReactFlow) | 节点编辑器核心——拖拽/连线/缩放 |
| UI组件 | **Ant Design 5** | 按钮/表单/表格/上传/提示 |
| Excel | **xlsx** (SheetJS) | 纯前端读取/写入 .xlsx |
| 状态管理 | **Zustand** | 轻量级全局状态（比Redux简洁90%） |
| 构建 | **Vite 6** | 秒级热更新开发 |

## 四、核心模块说明

### 4.1 Excel解析引擎 (`src/utils/excelParser.ts`)

**功能**：将 StoryFrame.xlsx / StoryGroup.xlsx / StoryBehavior.xlsx / Npc.xlsx 解析为结构化TypeScript对象。

**关键算法**：

```typescript
// 文件识别：通过文件名判断表类型
function detectTableType(filename: string): string | null

// 解析StoryFrame：跳过表头4行，从第5行起按列读取
function parseStoryFrame(rows: unknown[][]): StoryFrame[]

// 解析StoryGroup
function parseStoryGroup(rows: unknown[][]): StoryGroup[]

// 解析StoryBehavior（选项行为）
function parseStoryBehavior(rows: unknown[][]): StoryBehavior[]

// 解析Npc表构建名称→ID映射
function parseNpcTable(rows: unknown[][]): Record<string, number>

// 导出：StoryData → xlsx文件
function exportToExcel(frames, groups, behaviors): void
```

**StoryFrame 列号对照**：
| 列 | 字段 | 说明 |
|----|------|------|
| A(0) | frameId | 演出帧ID |
| B(1) | name | 名字 |
| C(2) | type | 1=对话 2=弹框大 3=全屏 5=弹框小 7=标题 |
| D(3) | groupId | 演出组ID |
| E(4) | nextFrame | 下一帧ID (空=末帧) |
| F(5) | isLastFrame | 是否末帧 |
| G(6) | behavior | 行为参数 |
| H(7) | picture | 留空 |
| I(8) | picture2 | **背景图路径** |
| J(9) | soundId | 音效ID |
| K(10) | text | 台词文本 |
| L(11) | additionalText | 性别变量 |
| M(12) | additionPar | NPC站位+表情 |
| N(13) | summary | 剧情梗概 |

### 4.2 流程图转换 (`src/utils/flowConverter.ts`)

将 StoryData 转为 ReactFlow 的 `nodes[]` 和 `edges[]`：

```
framesToNodes()  →  [{id, type:'storyNode', position:{x,y}, data:{...}}]
framesToEdges()  →  [{id, source, target, label, animated, style}]
```

**节点布局**：
- 同一 Group 的帧 → 水平排列（间距280px）
- 不同 Group → 垂直排列（间距160px）
- 每个 Group 使用不同颜色标识

**连线类型**：
- **NextFrame连线** → 灰色实线箭头（`stroke: #888`）
- **Behavior选项连线** → 橙色虚线箭头 + 选项标签（`stroke: #F5A623, animated: true`）

### 4.3 自定义帧节点 (`src/components/StoryNode.tsx`)

每个节点显示：
```
┌──────────────────────┐
│ [小明] 🔚    #6000102│  ← 角色 + 末帧标记 + FrameId
│ 所以我要去买菜！      │  ← 台词文本
│ 🖼BG 🔊音  🌿分支 📝文本│  ← 属性标签
│ ○──────────────○     │  ← Handle(连线锚点)
└──────────────────────┘
```

悬停时显示 Toolbar（FrameId + Group名）

### 4.4 编辑面板 (`src/components/FrameEditor.tsx`)

选中节点后，右侧出现编辑面板，可直接修改：
- 帧类型（下拉选择）
- 剧情组ID、下一帧ID
- 台词文本（多行输入）
- 背景图（下拉选择）
- 音效ID
- NPC站位参数（AdditionPar）
- 行为参数（选项分支）
- 剧情梗概

所有修改通过 `useStoryStore.updateFrame()` 实时同步到 Store，节点立即更新。

### 4.5 状态管理 (`src/store/useStoryStore.ts`)

```typescript
interface AppState {
  data: StoryData;           // 完整剧情数据
  loading: boolean;          // 加载状态
  selectedFrameId: number;   // 选中帧ID
  
  loadFiles(files);          // 加载Excel文件
  updateFrame(id, updates);  // 修改一帧
  addFrame(frame);           // 新增帧
  deleteFrame(id);           // 删除帧
  // ...
}
```

## 八、首次使用

1. 启动：`npm run electron:dev`
2. 点击「📂 加载文件夹」→ 选择包含 StoryFrame.xlsx / StoryGroup.xlsx / StoryBehavior.xlsx 的文件夹
3. 等待解析 → 流程图渲染
4. 点击任意节点 → 右侧编辑面板
5. 修改后点击「📤 导出Excel」→ 保存到文件
6. 将导出文件覆盖回原配置目录
